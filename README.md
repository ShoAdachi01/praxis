
### 1. **Nothing is screaming for attention**

* No focal UI fireworks
* No “look at me” elements
* Everything exists *because it needs to*, not because it can

### 2. **Nature is not decorative — it is structural**

* Stone = weight / commitment
* Moss = patience / time
* Wood = warmth / humanity
* Light = direction, not drama

This is crucial: Praxis shouldn’t *represent* nature.
It should **behave like nature**.

---

## Translate this directly into product design

### Design rule #1

**The app should feel like entering a space, not opening a tool.**

No “home screen” energy.
More like stepping onto tatami.

---

## Visual system (very concrete)

### Color palette (low contrast, grounded)

* Off-white / warm stone (background)
* Moss green (accent, very sparse)
* Charcoal / ink gray (text)
* Wood brown (secondary accent)

No pure white.
No pure black.
No saturated colors.

---

### Typography (quiet, human)

* One font family only
* Soft, slightly rounded
* Good at large sizes
* Comfortable with emptiness

Rules:

* Headlines are *never* bold
* Body text breathes
* Line height > font size ratio is generous

Text should feel like it belongs in a book, not a UI.

---

## Layout philosophy (this matters more than features)

### Rule of Asymmetry

Zen spaces are balanced, not symmetrical.

* One main element per screen
* Everything else recedes
* Asymmetry gives life without noise

---

### Rule of Space > Content

Whitespace is not “empty.”
Whitespace is **permission**.

On any screen:

* If adding something makes it feel fuller → don’t add it
* If removing something makes it calmer → remove it

---

## Interaction design (how users *move*)

This is where your concern about **commitment vs scrolling** gets resolved.

### Praxis should never scroll vertically

Scrolling = consumption
Praxis = intention

Instead:

* One screen
* One gesture
* One action

If the user scrolls, they’re already slipping back into Instagram mode.

---

## Core interaction mapped to the mood board

### Opening Praxis feels like:

* entering a quiet room
* shoes off
* no urgency

### The app does **one thing**:

> **Invites you to place a stone.**

Not literally — but conceptually.

---

## Reframing the core action using nature

Instead of:

* “Declare your bold action”
* “Commit to a goal”

Think:

> **“Place your next stone.”**

That stone:

* is small
* is deliberate
* changes the landscape slightly
* cannot be unplaced

This avoids nihilism **and** avoids hustle.

---

## Voice fits this *perfectly*

Typing feels like paperwork.
Voice feels like breath.

Design implication:

* Voice button is central
* Large
* Calm
* No waveform theatrics
* No transcription by default

You speak.
The stone is placed.
Silence follows.

---

## Feedback design (critical)

After committing, the app should respond with **nothing flashy**.

Something like:

* a subtle shift in light
* a single word:

  > “Placed.”
* or even just stillness

Then: done.

No “keep going.”
No “you got this.”

Zen trusts the user.

---

## What sharing becomes in this visual world

Sharing is not a feed.
Sharing is **co-presence**.

Think:

* you don’t see others’ stones
* you simply know others are also placing stones
* maybe a quiet indicator:

  > “3 stones placed today”

Like a garden you tend together, without talking.

---

## One sentence that should guide *every* design decision

> **If this were a physical space, would someone feel calmer after being here?**

If the answer is no, it’s not Praxis.
